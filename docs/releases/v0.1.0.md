# rss-agent-skills v0.1.0

Release date: 2026-05-21

`v0.1.0` is the first stable checkpoint for `rss-agent-skills` before splitting RSS workflows into multiple Skills.

## Highlights

- Ships the first portable Skill package: `skills/rss-ai-digest`.
- Supports OPML import, RSS 2.0 parsing, Atom parsing, filtering, scoring, dedupe, and source quality evaluation.
- Includes a curated base OPML with AI, engineering, security, product, and general technical sources.
- Adds deterministic source health tracking, failed-feed visibility, and source governance actions.
- Adds safe registry patching through `curate-sources` and `apply-source-patch`.
- Provides English and Chinese README files, contribution guidance, examples, release checklist, and MIT license.

## Stable Contract

This release stabilizes the single-Skill package shape:

```text
skills/rss-ai-digest/
├── SKILL.md
├── agents/openai.yaml
├── references/
└── scripts/rss_monitor.py
```

The CLI remains file-based and runtime-neutral. Agents should pass explicit registry, state, health, patch, and output paths.

## Validation

- `python3 -m unittest tests/test_rss_monitor.py -v`
- `quick_validate.py skills/rss-ai-digest`
- `git diff --check`

## Notes For Future Splits

Future packages such as `rss-source-curator`, `rss-alert-monitor`, `rss-digest-publisher`, and `rss-feed-discovery` should wrap shared RSS primitives rather than fork the current workflow into runtime-specific implementations.
